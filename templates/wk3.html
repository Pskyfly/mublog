第三周学习报告——李天航

最短路

最短路的应用十分常见，在不同的情况需要用到不同的算法。Bellman算法存储了所有的边穷举所有的三角不等式来进行松弛。，在这种方法的基础上进行改进后可以得到spfa算法，spfa算法可以处理边权为负的情况（所有算法都不能处理负环）。dijstra算法中每次得到的当前最短的路径，就是这个点的最短路，因为其他路径都比他长，不能被其他路径更新，然后用这个最短的路径来更新其他的点，可以用堆来优化，复杂度为O（nlogn）但是不能处理有负边的情况，但是在一些处理后依然可以使用（Johnson算法），弗洛伊德简单暴力地求出每个点到每个点的最短路。此外最短还能用记忆化深搜（dp），拓扑排序来求解。

动态规划（基础）

动态规划方法较多，需要多积累经验。思路是找到问题的子问题，然后对子问题初步进行扩展。中间可能可以用堆，二分，单调队列等方法进行优化。从前往后求解就是记忆化深搜，从而往前求解就是动态规划。

背包和状态压缩dp

背包的类型多种多样，代码也较为类似。01背包可以视为多阶段决策问题，因而需要二维数组dp[i][j],第一维表示考虑了前i个物品，第二维为背包容量。状态转移方程为

d p [ i ] [ j ] = m a x ( d p [ i − 1 ] [ j ] , d p [ i − 1 ] [ j − w [ i ] ] ] + v [ i ] )

如果j>背包容量，则

d p [ i ] [ j ] = d p [ i − 1 ] [ j ]

在枚举背包容量时，需要从大往小枚举皆可以。状态转移方程的意思是两种决策：我不要这个物品，那么此时的价值就和上一个物品在容量相同时的决策相同；或者我要这个物品，那么总价值背包容量减少后的价值加上这次的价值，两种取最大值。之所以01的原因是我对第i个物品做决策的状态都是由第i-1个物品转移而来，因而不会做重复的决策。

 

而完全背包则不一样，它的状态转移方程为

d p [ i ] [ j ] = m a x ( d p [ i ] [ j ] , d p [ i ] [ j − w [ i ] ] + v [ i ] )

而式子中的i变量都一样，所以可以去掉，简化为一维dp

d p [ j ] = m a x ( d p [ j ] , d p [ j − w [ i ] ] ] + v [ i ] )

也就是对第i个物品的决策可以由不同容量的第i个物品转移而来，因而每个物品可以选多个。但是注意这里的枚举的顺序必须是从前往后。如果是从后往前，需要用到的前面的值还没有求解出来，从而转换为了01背包。

而将他们综合起来就是就是混合背包问题，混合背包问题每个物品都有选择的次数上限P[i]（有的是1，有的是无限，有的是一个确定是数值），而即使是完全背包，选择次数也是有上限的，即背包容量/重量，01背包即为1。于是问题就被统一了起来，用一维的dp[j]代表背包容量是j时的最大价值。在枚举第i个物品，背包容量为j的同时，枚举每个物品的数量k，状态转移方程为

d p [ j ] = m a x ( d p [ j ] , d p [ j − k ∗ w [ i ] ] ] + k ∗ v [ i ] )

但是当每个物品数量很大的时候，复杂度会过高。所以我们可以进行二进制分组进行优化，这其实是一种用空间换取时间的优化方法。将每个物品的数量P[I]预处理为1+2+4+……+c（余数），将不同数量的物品视为不同的物品。也就是将原来重量为w，价值为v，数量为p的物品，分割为{w,v},{2w,2v},{4w,4v},……,{cw,cv}的物品，然后重新组成一个v，w数组，对其用01背包的状态转移方程即可dp[j]=max(dp[j],dp[j-w[i]]]+v[i])，但是这里是一维的dp，的背包容量必须从大到小枚举，可以想到从大开始的话,dp[j-w[i]]]还没有处理，因而当前物品只会有一个。优化后复杂度为O(nlogn)。二维数组的dp如果空间太大，可以用滚动数组来优化（涉及到计算的只需要两行）。状态压缩dp说难也不是很难，只是集结了每个性质有和没有的状态。一般形式为dp[n] [2^n]表示n个物品和n个物品的状态。

这是洛谷R54413654 ，和TSP问题差不多但是他最后不需要回到出发点（我一开始以为就是TSP，呜呜呜）这个题可以作为状压dp的模板。里面最容易错的是dp[j] [s] = min(dp[j] [s], dp[k] [s - (1 << (j - 1))] + dis[k] [j]);第一纬度的字母一定是不同的，否则两个子问题会相互包含。dp[j] [s] 大致意思是从j出发，经过{s}中的点后的最短距离，dp[k] [s - (1 << (j - 1))] + dis[k] [j]的意思是先从j走到k，然后从k出发，经过{s}中j以外的点后的最短距离，本质上也是进行了松弛操作。

区间dp和树形dp

区间dp……感觉就很玄学，状态转移方程什么形式的都有，只能多积累经验，常见的区间dp的方程为dp[i] [k]=max(dp[i] [k],dp[k] [j])也就是枚举区间的分割点，然后用二分前缀和之类的算法进行优化。树形dp的题目就比较规整（可以套板子），一般的形式为dp[i] [j]，i表示是第几个结点，j表示当前结点的状态，例如有或者没有，染什么颜色。在一开始对结点进行初始化，然后枚举它的子节点，对子节点进行递归搜索，其中会计算出子节点的dp值，然后将子节点的dp值统合到父节点中，最后综合根节点的所有状态就是所要的答案。

单调栈和单调队列

单调栈用于求解一个数左边或者右边最大最小的问题。维护单调栈，例如单调递增栈，如果当前结点比头结点小，则一直出栈直到栈顶元素小于当前元素，递减栈也同理，其中要不要加等号，还有一些边界问题需要具体题目具体讨论。单调队列可以用于解决定长区间最大最小值的问题，队头维护方法和单调栈一样，队尾要判断队尾元素是否在滑动窗口的外面。单调队列一般可以用于优化dp的问题，因为定长区间的最值问题在dp中比较常见，并且入队判断和出队判断都可以是不同的性质，较为灵活（比较难）。单调栈和单调队列中存放的是数组原元素的下标。后面一些斜率优化的题还需要继续学习。
