并查集和最小生成树

并查集用于解决不同集合之间的问题，而最小生成树用于求解不同集合连接起来的最短路径。如果有多个不同类比的集合则需要用到扩域并查集和权值并查集等。

最小生成树有两种算法，prim算法和卡鲁斯卡尔算法。

prim算法，prim算法需要用到邻接矩阵，时间和空间花费都比较大，因而Kruskal算法用得较多。

字符串哈希

字符串哈希可以将一个序列转换成一个数值存放于数组中，便于查询和统计数目。一维字符串可以用unordered_map来解决，用之前用reserve函数预分配空间可以加快处理的时间。此外还能将字符串视为一个P进制数，于是可以得到哈希值为a0p+a1p^2+……+an*p^n.而如果遇到二维矩阵或者是三维矩阵，unordered_map就无法处理，这时候需要选2个合适的哈希P值，比如9191891,6893911。

trie树

trie树也就是字典树，用于处理和字符串前缀相关的问题。它将字符串中的每个字母和每个结点进行编号，然后动态地将每一个结点按次序存入一个二维数组trie[N] [M]中N代表结点的序号，M代表可能出现的子节点数目。

插入操作：

xxxxxxxxxx

	

	

	

	

		

			

			

			
#define N 5000001

		

		
ll n, m;

		
int son[N][30], cnt[N], idx, flag;

		
int pre[N],sum[N];//pre用来记录前驱结点，sum用来记录当前子树的和

		
int insert(string s)

		
{

		
    int p = 0;//初始结点root

		
    bool flag1 = false, flag2 = false;//统计一些需要的信息

		
    rep(i, 0, s.length() - 1)//遍历每个字母，将它们插入到树中

		
    {

		
        int u = s[i] - '0';//获取一个新编号

		
//如果走到了同一个结点，也就是前缀相同，就继续走下去。

		
        if (!son[p][u])//如果树中没有我需要的结点则开辟一个新结点，赋予其一个新的编号

		
            son[p][u] = ++idx, flag1 = true,pre[idx]=p;//新的结点

		
        p = son[p][u];//走到子节点

		
        if (cnt[p] > 0) flag2 = true;

		
    }

		
    cnt[p]++;//一个字符串走完后，在最后一个结点上做标记，说明这里有个字符串

		
    while (p!=0)

		
    {

		
        sum[p] = cnt[p] + sum[son[p][0]] + sum[son[p][1]];//统计有多少字符串将p字符串作为前驱

		
        p = pre[p];

		
    }

		
    if (flag2 == true || flag1 == false)

		
        return true;

		
    else

		
        return false;

		
}

查询：

xxxxxxxxxx

	

	

	

	

		

			

			

			
int query(int l)//查询过程和插入类似，都要在树中走一般，只是不会改变数的结构

		

		
{

		
    int p = 0;

		
    ll ans = 0;

		
    rep(i, 1, l)

		
    {

		
        int u = s[i];

		
        if (son[p][u] == 0) return ans;

		
        p = son[p][u];

    }

}

kmp和AC自动机

kmp算法用于文本串和模式串的匹配。先对模式串进行预处理，找出各个长度的最长公共前后缀，将后缀的起始位置存入数组ne中，这其实也是加速匹配的基础。在模式串与文本串匹配的过程中已经匹配成功的长度为len，那么如果第len+1位匹配失败，则将模式串向前移动，使得当前长度的前缀移动至与后缀平齐也就是j=ne[j]，这样新进行匹配的长度就为前缀的长度，在此基础上继续匹配下一位。

倍增

倍增是加速模拟的一种算法，相当于对距离进行二进制分解，然后每次以2^k的距离接近目标使得其能达到加速的效果。最典型的应用有st表和lca（最近公共祖先）。

ST表

st表是为了求区间的最大值或者最小值。每次求一个区间时，就去求长度为1，2，4，8，……，2^k的区间最大值，而这些区间最大值可以通过预处理递推得到。st[i] [j]用来表示起点为i长度为2^j的区间最大值，对于任意一个i（当然，i+j-1<=n)

s t [ i ] [ j ] = m a x ( s t [ i ] [ j − 1 ] , s t [ i + j ] [ j − 1 ] )

意思是将长度为2^j的区间分成两个2^(j-1)的区间，然后分别求解。然后对于每一个区间的最大值st[i] [j]，可以取一个长度大于j/2的（也就是第一个小于j的）整数2^t（这个t也就是log2(j)向下取整，也可以预处理出来）然后分成左右两个区间进行求解，因为需要用到的区间（长度为1，2，4，8，……，2^k）已经提前预处理出来了，所以求解会很快，只需要O(1)的复杂度。求解时两个2^t区间会有重叠，但是不会影响我们求最值。（2^k在代码中用1<<k来表示）

rep(i, 1, n)

		

		
        st[i][0] = fr;//长度为2^0，也就是只有它本身

通过st表我们发现，其实倍增用到了分治的思想，巧妙地利用了2^k的相互关系，将他们作为子问题预处理出来，在具体问题求解时，化零为整，求得答案。当问题在2^k上有独特性质时，也可以用到st表的思想。

lca（最近公共祖先）

st表是区间上的性质，而lca是树上加速模拟的方法，在树上寻找两个结点的最短路径也是求最近公共祖先。求解最近公共祖先首先要将深度较高的结点移动与另一个结点深度相同的位置，然后两个结点一起移动，直到找到最近公共祖先。这里如果一步一步地移动，显然太慢了，于是我们想到如何能一次移动2^k步，第一步要移动至与另一个结点深度相同的位置，我们可以将他们的深度差设为h，对h进行二进制分解可以得到诸如110011……的式子，当它为1的时候我就移动2^k步,这样就只需要走log2（h）步就能到达，而第二步也是同理，如果2^k太大了，就判断2^(k-1)，还是太大了就判断2^(k-2)步直到2^0步。而每一个结点的第2^k父亲和深度可以通过dfs从根节点跑一遍得到。

预处理时也同样用到了lca中2^k的独特性质

f [ x ] [ i ] = f [ f [ x ] [ i − 1 ] ] [ i − 1 ]

意思是x结点的第2^i个父亲，是x结点第2^(i-1)个父亲的第2^(i-1)个父亲。

g [ x ] [ i ] = g [ x ] [ i − 1 ] + g [ f [ x ] [ i − 1 ] ] [ i − 1 ]

g[x] [i]代表从x+1个结点累加到x的第2^i个父亲结点的值，不包括x结点本身（因为不包含根节点root），所以这个公式的意思是从x+1个结点累加到x的第2^i个父亲结点的值等于，从x+1个结点累加到x的第2^(i-1)个父亲结点的值加上从x的第2^(i-1)+1个父亲结点累加到x的第2^i个父亲结点的值。

dfs预处理：

xxxxxxxxxx

	

	

	

	

		

			

			

			
int dep[N],f[N][32],g[N][32];//dep是深度，f是i的第2^j个父亲结点

		

		
void dfs(int x, int fa)

		
{

		
    dep[x] = dep[fa] + 1;//子结点是父亲结点深度加一

		
    f[x][0] = fa; g[x][0] = c[fa];//x的第2^0个父亲是fa，x的第2^0个父亲的值是fa的值

		
    for(ll i=1;(1<<i)<=dep[x];i++)//从足够大的地方开始预处理

		
        f[x][i] = f[f[x][i - 1]][i - 1], g[x][i] = g[x][i - 1] + g[f[x][i - 1]][i - 1];

		
    for (int i = head[x]; i != -1; i = ne[i])

		
    {

		
        int v = edges[i];//在树上进行搜索

		
        dfs(v, x);

		
    }

		
}

xxxxxxxxxx

	

	

	

	

		

			

			

			
int lca(int x, int y)

		

		
{

		
    if (dep[x] < dep[y]) swap(x, y);//确保x是深度较大的结点

		
    repp(i, 19, 0)

		
    {

		
        if (dep[fa[x][i]] >= dep[y]) x = fa[x][i];//二进制分解，从高位到低位逐位比较更新，如果更新到第2^i个父亲深度仍然太大则更新，否则就代表已经到达了终点

		
    }

		
    if (x == y) return x;

		
    repp(i, 19, 0)

		
    {

		
        if (fa[x][i] != fa[y][i]) x = fa[x][i], y = fa[y][i];//两个结点一起更新

		
    }

		
    return fa[x][0];//得到的答案是x的直接父亲

		
}

（2）用f[i] [j] = f[f[i] [j - 1]] [j - 1]预处理每个跳转的位置

（3）用二进制接近法，同时统计答案，公式为：

i f ( c h e c k ( ) == f a l s e ) x = f a [ x ] [ i ]

我们最终的目的是让check成立，所以当它不满足时代表还没有到达终点，需要继续用二进制的方法接近终点。check里需要比较的应当都是下一个状态的相对关系。例如如果想要快速得到第一个让fa[x] [i]>=a的i，就需要用到if(fa[x] [i]<a) x=fa[x] [i]。
